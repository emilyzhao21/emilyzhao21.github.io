重装Git后，Git bash Here 和 Git GUI here 失效的解决方法

发表于 2015-10-24 | 分类于 疑难问题

今天手贱的重装了最新版的Git，结果鼠标右键菜单里面的Git Bash Here和Git GUI Here 失效了(后来才发现只是在桌面上无法显示在文件夹中右键打开还是可以的)

主要原因就是:注册表里面的错误,command中的数值数据追加了一些东西

win+R打开运行对话框，输入regedit（博主习惯性的老是输入register，哈哈~~）打开注册表，找到

  1. HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Directory\background\shell\git_gui\command，修改数据，一般都是后面多加了一些东西，去掉就行了，改成例如：”E:\web\github\Git\cmd\git-gui.exe” 修改为你自己的路径就行了
  2. HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Directory\background\shell\git_shell\command 改成例如：”E:\web\github\Git\git-bash.exe”修改为你自己的路径就行了
